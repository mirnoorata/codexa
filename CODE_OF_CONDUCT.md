# Code of Conduct

This is a short, plain-language Code of Conduct. The intent is to make the
project a useful place to collaborate without spelling out every possible
interaction.

## What I expect from contributors

- **Be direct.** Say what you mean. Technical disagreements are not rudeness.
- **Be useful.** Framing, reproductions, and suggested fixes are more welcome
  than demands.
- **Treat the maintainer as a person.** I work on this around other things.
  Escalating in tone because a response is slow is not how this gets faster.
- **No harassment, no personal attacks, no discriminatory language** — in
  issues, PRs, reviews, or any other project surface. That includes "joking"
  versions.

## What contributors can expect from me

- **Honest reasons when I decline a change**, even if the reason is "not in
  scope for a project I maintain alone."
- **Credit where credit is due** — PRs that ship keep their commit attribution,
  and security reports get advisory credit when the reporter wants it.
- **No silent treatment on serious reports.** Security advisories and
  code-of-conduct reports always get a reply, even if the reply is "I am
  looking into this and will follow up."

## If something goes wrong

If you experience or witness behavior that violates the expectations above:

1. If it is a security issue, follow [SECURITY.md](SECURITY.md) instead.
2. For everything else, contact the maintainer through
   [@mirnoorata](https://github.com/mirnoorata)'s GitHub profile or, if
   appropriate, via a private security advisory on this repository.

Possible responses include: a private note, a public clarification, editing or
locking a thread, temporary or permanent exclusion from project spaces. The
maintainer decides.

## Out of scope

This Code of Conduct covers project spaces — this repository's issues, PRs,
discussions, commit messages, and any official channels. It does not attempt
to police behavior elsewhere on the internet.
